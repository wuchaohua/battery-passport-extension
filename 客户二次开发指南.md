# 电池护照平台 - 客户二次开发指南

适用场景: 在不获取 Java 后端源码的情况下进行业务定制

---

## 六种定制方式

### 1. SPI 插件机制

实现平台公开的接口，打包为 jar 提交给运维热部署。

可实现的接口:
- BatteryDataValidator: 自定义电池数据校验
- BatteryDataTransformer: ERP/MES 数据格式转换
- ReportCustomizer: 报告内容定制
- NotificationInterceptor: 通知拦截和定制
- EventSubscriber: 事件订阅

开发步骤:
1. 在 pom.xml 中依赖 battery-plugin-api
2. 实现一个或多个接口
3. 在 META-INF/services/ 中注册实现类
4. 编译为 jar
5. 提交运维放入 plugins/ 目录，平台自动热加载

### 2. RESTful API + SDK

主要 API:
- GET /api/v1/passports - 护照列表
- GET /api/v1/passports/{id} - 护照详情
- POST /api/v1/passports - 创建护照
- GET /api/v1/reports/{id}/download - 下载报告
- POST /api/v1/webhooks - 注册 Webhook

### 3. Webhook 事件回调

事件类型: PASSPORT_CREATED, PASSPORT_UPDATED,
REPORT_GENERATED, QUALITY_ALERT, TRACE_UPDATED

### 4. 脚本引擎扩展

Groovy/JS 脚本热加载，无需编译打包。
用于快速规则调整、报告格式定制。

### 5. 前端远程组件

React 组件通过配置注入:
- passport.dashboard.top
- global.header.right
- home.widget.left

### 6. 配置化定制

通过 yaml 覆盖:
- battery.sso.provider - SSO 协议选择
- battery.validation.* - 校验阈值
- custom-branding.* - 品牌色和 Logo

## 插件开发规范

- IPC 接口版本与平台主版本绑定
- 每个插件使用独立 ClassLoader 隔离
- 校验器执行时间不应超过 500ms
- 插件禁止启动新线程或定时任务
- 抛异常会被平台捕获，不影响主流程

## FAQ

Q: 字段校验要改，每次都要平台发版吗？
A: 不需要。通过 Groovy 脚本引擎或 SPI 插件热部署即可。

Q: 前端加自己的页面怎么办？
A: 平台支持远程组件加载，将 React 组件发布到 CDN 并注册即可。

Q: 买了源码交付版本，上述方式还适用吗？
A: 适用。建议走 SPI 插件机制以便平台升级时无需大幅调整。
